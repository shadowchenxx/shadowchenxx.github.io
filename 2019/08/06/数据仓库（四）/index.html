hive表分类

1.内外部表

  • 管理表（内表）

    由hive.site.xml文件中属性hive.metastore.warehouse.dir所定义目录的子目录之下。当我们删除一个管理表时，hive也会删除这个表中的数据。

  • 外部表

    建表时由关键字external决定，不再使用👆配置文件中的路径，用location指定。因为表是外部的，在对外部表执行删除操作时，只是删除掉描述表的元数据信息，并不会删除数据。

    用describe formatted tablename语句可以看出来是否时外部表

补充1:利用like关键字复制表结构时，若未指定表时内表还是外部表时。由被复制表的属性决定

2.分区表

🌧️其他数据库类似，hive 中也有分区表的概念。分区表的优势体现在可维护性和性能上面，可以理解为它就是一个过滤器。

补充2:数据仓库中，一种常用的安全措施时将hive设置严格mapred模式，这样如果对分区进行查询而where条件中未限制，将会禁止提交这个查询。

2.1向分区表中装载数据

注意的几点：限制资源使用的属性

属性名称 默认值 描述
hive.exec.dynamic.partition false 设置true，开启动态分区功能
hive.exec.dynamic.partition.mode false 设置constrict，允许所有分区都是动态的
hive.exec.max.dynamic.partitions.pernode 100 每个mapper或reducer可以创建的最大动态分区个数
hive.exec.max.dynamic.partitions 1000 一个动态创建dml语句可以创建的最大动态分区个数
hive.exec.max.created.files 100000 全局可以创建的最大文件数，

补充3:

  • 除日期维表外，其他表都使用orc文件格式，并设置表属性支持事务
  • 日期维表只会追加数据而不更新
  • hive中并没有主键、外键、唯一性约束、非空约束这些关系型数据库的概念。
