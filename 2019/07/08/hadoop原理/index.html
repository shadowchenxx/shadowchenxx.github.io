hadoop离线开发架构和原理
2019-07-08

概述

离线数据平台是整个数据平台的根本和基础。一般情况下，数据相关人员很少通过直接写mapreduce代码来处理数据，而是通过hql，但这是远远不够的。要想走的远，必须理解hadoop的知识，包括概念、框架、原理、执行过程等

组件

1.hadoop的优缺点

  • 优点

    a.处理超大文件

    b.运行在廉价的机器上

    c.高容错性和高可靠性

    d.流式访问数据

  • 缺点

    a.不适合低延迟数据访问

    b.无法高效存储大量小文件

    c.不支持多用户的随机读写

2.mapreduce的优缺点

  • 优点

    a.已于编程

    b.良好的扩展性

    c.高容错性

  • 缺点

    a.应用场景局限：实时计算、流计算、DAG计算

3.架构
  • hadoop：namenode + datanode
  • mapreduce : client + JobTracker + TaskTracker + Task(mapTask,ReduceTask)
4. mapreduce 编程
5.任务提交流程
  • 客户端提交一个job，jobtracker会给这个job一个id值
  • jobtracker会根据输入文件计算输入分片（input split），分配资源，做初始化（创建一个job对象，以便后面jobtracker监控）
  • 每个分片创建一个map任务，根据tasktracker的忙闲情况分配map任务和reduce任务
  • 通过心跳机制监控tasktracker的状态和进度
  • 当jobtracker获得了最后一个完成指定任务的tasktracker，jobtracker将该任务置为成功
6.mapreduce内部执行原理详解

  • 输入分片

    实际处理中应将小文件进行合并，否则如果输入多个文件而且文件大小均远小于块大小，会导致生成不必要的map任务，这也是mapreduce优化计算的一个关键

  • map

    key_value

  • combiner

    combiner阶段可选择执行，是一种本地化的reduce，可以优化重复输出

    ⚠️ 存在风险

    使用combine操作原则上不会影响最后的取值（如汇总、最大值、最小值），但是平均值就会引起出错。

  • shuffle

    • map端shuffle
      • 分区（partition）、sort（排序）、合并（combine）、归并（merge）
    • reduce端shuffle
      • copy、merge、reduce task 输入

  • reduce

    • reduce任务调用reduce函数

总结

HDFS和Mapreduce是hive底层的技术，在使用hive的过程中，必须理解。🐶
