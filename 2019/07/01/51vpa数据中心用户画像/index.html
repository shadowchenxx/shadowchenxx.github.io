一、系统功能设计

  • 用户画像系统基于各软件接口，使用真实的用户数据。数据的爬取、更新、维护是系统的关键

  • 用户画像

    • 单个分析:基于单个案例的分析，提供细粒度的证据支持
    • 集体分析:居于一批用户的分析，挖掘用户之间的共同特征

  • 查询接口：介入系统的多少，是衡量一个系统成败的关键，需要提供一个友好的查询接口

1.模块设计

1.1 数据爬虫

  • 申请接口开发
  • 限制抓取频次，防止接口被封
  • 抓取数据，清洗之后进入临时库（hive_ods）

1.2 个人画像

  • 给用户打标签（基础标签和统计标签）{k-means 聚类}

    K-means 聚类算法一般用户 n 维连续空间中的对象，使用K-means算法之前需要制定初始的K值，K 值的选定可以根据经验也可以根据簇 的内聚程度和簇之间的分离程度打分判断。由于本系统分析的数据的特点，初始质心的选择采用随机选择方式，多次聚类计算之后，选择 SSE(误差的平方和)最小的簇集。

  • 分析用户行为并进行预测（决策树、随机森林、knn、神经网络等算法）

  • 协同过滤推荐

1.3 集体画像

  • 引入可视化模块，供olap分析使用

1.4 查询接口

  • 数据的及时性（引入spark）
